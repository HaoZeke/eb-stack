* Top-level usage

#+begin_src text
eb-stack {solve|parse|check-recipe|solve-json|sbom|bump|mcp|ingest} [subcommand options]
#+end_src

Built on [[https://docs.rs/clap][clap]]'s derive API. Every subcommand accepts
~--help~ for its own usage text; ~eb-stack --version~ prints the crate
version.

Subcommands at a glance:

| Subcommand | Role |
|------------|------|
| ~solve~ | Parse trees, resolvo co-select, lock + optional SBOM / build list / stack diff |
| ~parse~ | Dump parsed candidates as JSON (no solve) |
| ~check-recipe~ | One recipe: robot dep completeness + packaging gates (*resolves* rung) |
| ~solve-json~ | Solve from a pre-built universe JSON |
| ~sbom~ | CycloneDX from an existing lock |
| ~bump~ | Next-generation easyconfig rewrite (hierarchy-aware auto deps) |
| ~mcp~ | Stdio MCP server (~eb_check_recipe~ / ~eb_bump~ / ~eb_solve~) |
| ~ingest~ | Foreign conda-forge / Spack recipe → EB scaffold |

* solve

Parse a tree of ~.eb~ files, solve with resolvo, write a lock and a planned
CycloneDX SBOM. This subcommand is the primary entry point.

#+begin_src text
eb-stack solve
  --easyconfigs PATH...
  --policy PATH
  [--baseline-easyconfigs PATH]
  [--baseline-toolchain-version VERSION]
  [--lock-out PATH]              (default: stack.lock.json)
  [--sbom-out PATH]
  [--build-list-out PATH]
  [--stack-diff-out PATH]
#+end_src

- ~--easyconfigs~ :: Root directory of easyconfigs, walked recursively for
  ~*.eb~. Required, repeatable: pass it more than once to layer trees, with
  *later* paths overriding earlier ones for the same
  name+version+toolchain identity (a site overlay on top of an upstream
  checkout). See
  [[file:../howto/solve-lock.org][how to layer a site overlay]].
- ~--policy~ :: Policy JSON path. Required. See the
  [[file:policy.org][policy schema]].
- ~--baseline-easyconfigs~ :: Optional second tree (or the same tree) used to
  build the baseline lock for upgrade comparisons and ~require_upgrade~. When
  omitted, no baseline is built and ~--stack-diff-out~ is an error.
- ~--baseline-toolchain-version~ :: When the baseline tree contains
  *multiple* generations of the policy's toolchain family, this pins which generation is
  the baseline. Without it, the nearest lower generation than the policy
  target is chosen automatically (EasyBuild-style version ordering, e.g.
  target ~2025b~ with ~2024b~ and ~2025a~ present in the tree picks
  ~2025a~, not ~2024b~).
- ~--lock-out~ :: Where to write the solved lock. Default ~stack.lock.json~.
- ~--sbom-out~ :: Where to write the planned CycloneDX SBOM. *Opt-in*: with
  no ~--sbom-out~, no SBOM file is written at all -- there is no default
  filename. Pass it explicitly to get an SBOM alongside the lock.
- ~--build-list-out~ :: Optional plain-text build list, one selected
  easyconfig path per line, dependencies before the packages that need them.
- ~--stack-diff-out~ :: Optional markdown diff of the solved lock against the
  baseline lock. Requires a baseline (see ~--baseline-easyconfigs~).

* check-recipe

Packaging check for *one* easyconfig: resolve every runtime and build
dependency against robot tree(s), and run packaging gates (positional
checksum lint, required ~configopts~ substrings). This establishes only the
*resolves* rung of the reporting ladder -- not builds, not binary
verification. See [[file:../explanation/lifecycle.org][lifecycle]] and
[[file:../howto/run-annual-bump.org][annual bump]].

Cross-toolchain deps (for example ~quill~ on ~GCCcore~ while the recipe is
~gfbf~) are matched by identity, not by filtering the universe to the
recipe's toolchain alone. Exit 1 if any dep is missing or a packaging gate
fails.

#+begin_src text
eb-stack check-recipe
  --recipe PATH
  --easyconfigs PATH...
  [--require-configopt SUBSTR]...
  [--metadata-only]
  [--scaffold-missing DIR]
#+end_src

- ~--recipe~ :: Path to the ~.eb~ file to validate. Required.
- ~--easyconfigs~ :: Robot easyconfig tree(s), walked recursively. Required,
  *repeatable*: later paths override earlier ones for the same
  name+version+toolchain identity (draft overlay on upstream). Same
  precedence rule as ~solve~.
- ~--require-configopt SUBSTR~ :: Assert that ~configopts~ contains this
  substring. Repeatable. Used to gate product flags (e.g.
  ~-Dwith_metatomic=true~, ~-DQMC_MPI=ON~).
- ~--metadata-only~ :: Skip the missing-dep robot check; only run packaging
  metadata gates (checksum structure, required configopts, etc.). Useful
  when the robot tree is absent and you only need the packaging lint.
- ~--scaffold-missing DIR~ :: Write draft companion ~.eb~ files for every
  missing dep under ~DIR~ using the EasyBuild letter/name layout
  (~e/eOn/...~, ~q/quill/...~). The packaging prep loop for stacks that
  need greenfield companions: scaffold, fill sources/checksums, re-run
  ~check-recipe~ with the draft tree as a later ~--easyconfigs~ overlay.

Missing-dep output names generations where the package *does* exist
("available at other generations: ..."); treat that list as the work queue.
A ~[packaging]~ checksum finding means reorder ~checksums~ (all sources
first, then patches) -- never bypass the gate.

* parse

Parse easyconfigs and print JSON candidates. A debug / universe dump: no
solving happens.

#+begin_src text
eb-stack parse
  --easyconfigs PATH
  [--toolchain-name NAME]
  [--toolchain-version VERSION]
#+end_src

- ~--easyconfigs~ :: Root directory, walked recursively. Required.
- ~--toolchain-name~, ~--toolchain-version~ :: Both together filter the
  printed candidates to one toolchain generation. Either alone has no
  filtering effect (both must be present to filter).

Prints the parsed [[https://docs.rs/serde_json][serde_json]]-pretty array of
candidates to stdout: ~name~, ~version~, ~toolchain~, optional
~versionsuffix~, ~easyconfig_path~, ~dependencies~, ~builddependencies~.

* solve-json

Legacy entry point: solve from an already-built universe JSON instead of parsing
~.eb~ files. Still exercised by the test suite; useful for driving the
solver from hand-written or externally generated candidate data.

#+begin_src text
eb-stack solve-json
  --universe PATH
  --policy PATH
  [--baseline PATH]
  [--lock-out PATH]              (default: stack.lock.json)
  [--sbom-out PATH]
  [--build-list-out PATH]
  [--stack-diff-out PATH]
#+end_src

- ~--universe~ :: Universe JSON path (the ~parse~ output shape, wrapped with
  a ~toolchain~ and optional ~generation_label~; see
  [[file:formats.org][format reference]]). Required.
- ~--policy~ :: Same policy schema as ~solve~. Required.
- ~--baseline~ :: Optional baseline *lock* JSON (not an easyconfig tree) used
  the same way ~--baseline-easyconfigs~ is on ~solve~.
- ~--sbom-out~ :: Same opt-in behavior as ~solve~: omit it and no SBOM file
  is written.
- Remaining flags are identical to ~solve~ (except ~--universe~ is a single
  pre-built JSON file, not a repeatable ~--easyconfigs~ tree -- overlay
  merging happens before this command runs, when the universe JSON is
  built).

* sbom

Emit a CycloneDX SBOM from an already-solved lock file, without re-solving.

#+begin_src text
eb-stack sbom
  --lock PATH
  [--out PATH]                    (default: stack.cdx.json)
#+end_src

- ~--lock~ :: Path to a ~stack.lock.json~. Required.
- ~--out~ :: Where to write the CycloneDX document. Default
  ~stack.cdx.json~.

This path uses lock-level co-stack edges (every package depends on every
other co-selected package) rather than the declared per-package dependency
edges ~solve~ / ~solve-json~ can attach when they still have the universe in
hand. See [[file:formats.org][the SBOM format reference]].

* bump

Produce a next-generation easyconfig from an existing recipe. See the
[[file:../tutorial.org][tutorial]] for a full worked example and
[[file:../howto/bump-recipe.org][the how-to]] for override syntax.

#+begin_src text
eb-stack bump
  --source PATH
  --toolchain-name NAME
  --toolchain-version VERSION
  [--version VERSION]
  [--source-checksum SHA256]
  [--dep NAME=VERSION]...
  [--easyconfigs DIR]
  [--hierarchy-fixture PATH]
  [--keep-old-deps]
  [--out-dir PATH]
  [--out PATH]
#+end_src

- ~--source~ :: Source ~.eb~ file to rewrite. Required.
- ~--toolchain-name~, ~--toolchain-version~ :: Target toolchain. Always
  rewritten. Both required.
- ~--version~ :: New application version. Omit to keep the source's
  ~version = '...'~ unchanged (the common case: same release, new
  toolchain).
- ~--source-checksum SHA256~ :: New sha256 for the source tarball. Applies
  together with ~--version~ (the tarball name changes with the version).
  When the version changes and this is omitted, the source
  checksum entry's key is still renamed to the new versioned tarball name,
  but the checksum value is left stale and a warning is printed. See
  [[file:../howto/bump-recipe.org][updating the source checksum]].
- ~--dep NAME=VERSION~ :: Dependency version override, repeatable. Applies
  to matching tuples in both ~dependencies~ and ~builddependencies~. See
  [[file:../howto/bump-recipe.org][operator preservation rules]]. When
  ~--easyconfigs~ is also set, a ~--dep~ override wins over the
  auto-resolved version for that name.
- ~--easyconfigs DIR~ :: Directory of easyconfigs (walked recursively, same
  parser as ~solve~) used to auto-resolve every ~dependencies~ /
  ~builddependencies~ version for the target toolchain generation.
  Resolution is hierarchy consensus then resolvo joint SAT (hierarchy
  pins as exact requirements on a synthetic root; see
  [[file:../howto/bump-recipe.org][bump how-to]] and
  [[file:../explanation/architecture.org][architecture]]). Omit to fall back
  to hand ~--dep~ only. See [[file:../tutorial.org][the tutorial]] for the
  zero-hand-fed flow.
- ~--hierarchy-fixture PATH~ :: Optional EasyBuild ~get_toolchain_hierarchy~
  capture (JSON) that overrides hierarchy discovery. Rarely needed: with
  ~--easyconfigs~, GCC-family generations (~foss~/~gompi~/~gfbf~) derive
  their hierarchy from the tree's own toolchain definitions, bare
  ~GCCcore-x.y.z~ targets synthesize ~[SYSTEM, GCCcore]~, and
  ~foss-2023b~/~2024a~/~2025a~ have built-in fixtures. The explicit fixture
  is the escape hatch for non-GCC toolchains (~intel~, ~nvidia-compilers~).
  Applies only with ~--easyconfigs~.
- ~--keep-old-deps~ :: When set, a dependency ~--easyconfigs~ cannot resolve
  keeps its source version (with a warning) instead of failing the bump.
  Default: unresolved dependencies fail the bump with a non-zero exit
  naming the dependency and the target toolchain. A ~versionsuffix~-pinned
  dependency is never bumped regardless of this flag.
- ~--out-dir~ :: Output directory (created if missing); file is written
  under the conventional
  ~{name}-{version}-{toolchain_name}-{toolchain_version}.eb~ basename.
  Conflicts with ~--out~.
- ~--out~ :: Explicit output file path, any name. Conflicts with
  ~--out-dir~. With neither flag, the file is written to the current
  directory under the conventional basename.

Every field not named above -- ~easyblock~, ~homepage~, ~description~,
~toolchainopts~, comments, and any dependency neither ~--dep~ nor
~--easyconfigs~ auto-resolve changed -- is
copied through byte-for-byte from the source.

* mcp

Serve the packaging workflow as MCP tools over *stdio* (JSON-RPC 2.0). No
extra flags: the process reads requests from stdin and writes responses to
stdout until EOF.

#+begin_src text
eb-stack mcp
#+end_src

Register with a harness that speaks MCP, for example:

#+begin_src bash
claude mcp add eb-stack -- eb-stack mcp
# harness-equivalent: pass `eb-stack mcp` as the stdio server command
#+end_src

Tools exposed (same semantics as the matching CLI subcommands):

| Tool | CLI analogue | Required arguments |
|------+--------------|--------------------|
| ~eb_check_recipe~ | ~check-recipe~ | ~recipe~, ~easyconfigs~ (array) |
| ~eb_bump~ | ~bump~ | ~source~, ~toolchain_name~, ~toolchain_version~ |
| ~eb_solve~ | ~solve~ | ~easyconfigs~ (array), ~policy~ |

Optional arguments mirror the CLI:

- ~eb_check_recipe~: ~require_configopts~ (array of strings),
  ~scaffold_missing~ (directory path). Note: the CLI ~--metadata-only~ flag
  is not currently mirrored on the MCP tool; use the CLI for metadata-only
  packaging lint.
- ~eb_bump~: ~version~, ~source_checksum~, ~deps~ (object name→version),
  ~easyconfigs~, ~hierarchy_fixture~, ~keep_old_deps~, ~out_dir~, ~out~.
- ~eb_solve~: ~baseline_easyconfigs~, ~baseline_toolchain_version~,
  ~lock_out~, ~sbom_out~, ~build_list_out~, ~stack_diff_out~.

Every ~eb_check_recipe~ result embeds the three-claim ladder state and
derived next-actions (missing-dep work queue, packaging fixes). Tool
failures are returned as MCP ~isError: true~ content, not as bare JSON-RPC
errors, per the MCP tools/call contract. Library entry point:
[[file:rust-api.org][Rust API]] module ~mcp~.

* Exit codes

| Code | Meaning                                                          |
|------+-------------------------------------------------------------------|
|    0 | Success                                                          |
|    1 | Error: parse failure, unsatisfiable policy, missing dep, packaging gate failure, missing file, or rewrite failure. The message names real package/version/field data, not internal solver identifiers. |

* ingest

Convert a foreign package recipe into a parseable EasyBuild scaffold.

#+begin_src text
eb-stack ingest
  --source PATH
  [--format auto|conda-forge|spack]
  [--toolchain-name foss]
  [--toolchain-version 2024a]
  [--out PATH | --out-dir DIR]
#+end_src

- ~--source~ :: ~meta.yaml~ / ~recipe.yaml~ (conda-forge) or ~package.py~
  (Spack). Required.
- ~--format~ :: ~auto~ (default, from filename), ~conda-forge~, or ~spack~.
- ~--toolchain-name~ / ~--toolchain-version~ :: scaffold toolchain identity
  (operator-supplied generation; not invented from the foreign recipe).
- ~--out~ / ~--out-dir~ :: output path or letter/name layout directory.

The emit path is honest: name, version, source URL/filename, sha256 when
present, multi-source layout, static ~configopts~ when the foreign DSL
states them, and dependency *names* come from the foreign input. Product
flags, patches, and hand-maintainer pins remain residual (header warnings).

**Not a landable PR.** Finish with EasyBuild (~inject-checksums~,
~check-contrib~, ~eb --robot~) and residual judgment per
~skills/new-package/SKILL.md~. Do not hardcode product ~configopts~ into
this tool.

- ~--easyconfigs DIR~ :: Robot easyconfig tree(s). Same two-stage resolve as
  ~bump~ (hierarchy consensus + resolvo joint pins). Repeatable; later wins
  on identity conflict. Fills generation-native dep versions and preferred
  package-name casing.
- ~--keep-old-deps~ :: Keep residual foreign versions when the robot has no candidate.
- ~--hierarchy-fixture PATH~ :: Optional hierarchy JSON override.

Without ~--easyconfigs~, dep versions stay residual (foreign floors or
~0.0.0~). With it, expect warnings naming ~hierarchy consensus~ and
~resolvo joint~. Fixtures: ~fixtures/foreign_ingest/~.

- ~--residual-queue PATH~ :: optional residual work-queue JSON path. Default:
  write ~{stem}.residuals.json~ next to the scaffold. Items classify dep
  version / product_config / moduleclass / sanity / checksum residuals for
  a local-ai agent — not product ~-D~ defaults invented by ~eb-stack~.
