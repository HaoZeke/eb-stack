The ~eb_stack~ crate is the library behind the ~eb-stack~ binary. CLI
subcommands and MCP tools call the same public paths documented here.

The module tree below is *generated from the crate source* (rustdoc comments)
via ~sphinx-rustdocgen~ and rendered with ~sphinxcontrib_rust~ during
~pixi run -e docs docbld~. Expand "Crate eb_stack" in the sidebar, or follow
the injected toctree, for every public item.

* Public modules (map)

| Module | Role |
|--------+------|
| ~domain~ | Core types: toolchain, candidate, lock, policy |
| ~eb_parse~ | Restricted-Python easyconfig parse, ~check_recipe_deps~, ~packaging_gate~ |
| ~eb_emit~ | ~emit_next_generation~; auto-resolve = hierarchy + resolvo joint pins |
| ~hierarchy~ | Sub-toolchain membership and generation-consensus floors |
| ~select~ / ~resolvo_provider~ | CDCL SAT co-selection; ~resolvo_resolve_dep_versions~ for bump/ingest |
| ~report~ | Build list and stack-diff markdown |
| ~sbom~ | Planned CycloneDX 1.5 via ~cyclonedx-bom~ (~lock_to_bom~, …) |
| ~foreign~ | conda-forge / Spack parse → EB scaffold (+ optional robot resolve) |
| ~mcp~ | JSON-RPC MCP tool surface (~eb_check_recipe~, ~eb_bump~, ~eb_solve~) |
| ~version~ | EasyBuild-style version ordering |

* Stability

The CLI and MCP tool argument schemas are the supported operator contracts
through 0.x. Library item paths may move before 1.0; pin the crate version
when embedding.
