* What it is

  ~eb-stack~ parses a tree of [[https://docs.easybuild.io/][EasyBuild]] easyconfig (~*.eb~) files, co-selects a
  consistent software stack for one toolchain generation with a CDCL SAT
  solver ([[https://github.com/mamba-org/resolvo][resolvo]]), and writes the result as a machine-readable lock file plus an
  opt-in planned CycloneDX SBOM. It also rewrites an existing easyconfig into
  the next toolchain generation (~bump~), auto-resolving every dependency and
  build-dependency version from a directory of easyconfigs (hierarchy-aware
  across ~GCCcore~/~GCC~/~gfbf~/~gompi~/... sub-toolchains) with no
  hand-fed ~--dep~ flags required, and reports the selected build order and
  a baseline-vs-solved stack diff for pull request review.

* What it is not

  - *Not* an EasyBuild replacement. ~eb-stack~ reads and rewrites easyconfig
    text; it does not build software and does not implement EasyBuild's
    build-step framework.
  - *Not* a general-purpose Python dependency resolver. The domain model is
    EasyBuild's ~name~ / ~version~ / ~toolchain~ / ~dependencies~ shape only.
  - *Not* a post-install compliance scanner. The emitted SBOM is a *planned*
    inventory from the lock, produced before anything is built.

* Install

  Build from source with Cargo:

#+begin_src bash
git clone https://github.com/HaoZeke/eb-stack.git
cd eb-stack
cargo test --locked
cargo build --release
# binary: target/release/eb-stack
install -Dm755 target/release/eb-stack ~/.local/bin/eb-stack
#+end_src

  Verify:

#+begin_src bash
eb-stack --version
#+end_src
